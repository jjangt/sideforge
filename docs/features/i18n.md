# 다국어(i18n) 가이드

## 기술 스택

- `i18next` + `react-i18next` — 번역 엔진
- `expo-localization` — 기기 언어 감지

## 지원 언어

| 코드 | 언어 | 기본 |
|------|------|------|
| `ko` | 한국어 | ✅ (fallback) |
| `en` | 영어 | |
| `ja` | 일본어 | |
| `zh` | 중국어 (간체) | |

## 파일 구조

```
src/i18n/
├── index.ts          # 초기화 (자동 import in _layout)
└── locales/
    ├── ko.ts         # 한국어
    ├── en.ts         # 영어
    ├── ja.ts         # 일본어
    └── zh.ts         # 중국어
```

## 사용법

### 컴포넌트에서

```tsx
import { useTranslation } from 'react-i18next';

function MyComponent() {
  const { t } = useTranslation();
  return <Text>{t('landing.title')}</Text>;
}
```

### 번역 키 규칙

```
{페이지}.{요소}
```

예시:
- `landing.cta` — 랜딩 페이지 CTA 버튼
- `onboarding.interests` — 온보딩 관심사 질문
- `common.next` — 공통 "다음" 버튼

### 언어 변경 (프로그래밍적)

```tsx
import i18n from '../src/i18n';

i18n.changeLanguage('en');
```

## 새 언어 추가 방법

1. `src/i18n/locales/{code}.ts` 파일 생성 (기존 파일 복사 후 번역)
2. `src/i18n/index.ts`에 import + resources에 추가
3. supported 배열에 코드 추가

## 새 번역 키 추가 방법

1. `ko.ts`에 키 추가 (한국어가 기본)
2. 나머지 언어 파일에도 동일 키 추가
3. 컴포넌트에서 `t('section.key')` 사용

## 주의사항

- 번역 파일은 `as const`로 타입 안전성 보장
- fallback은 항상 한국어(`ko`)
- 기기 언어가 지원 언어에 없으면 자동으로 한국어
