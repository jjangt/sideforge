# Phase 2 기능 기획

## 개요

Phase 1(YouTube 분석 MVP) 이후, 상용화 수준의 서비스로 개선하기 위한 기획.

---

## 1. 플랫폼 선택 UI

### 현재 문제
- 분석 페이지가 YouTube 전용처럼 보임
- placeholder, 설명 모두 YouTube만 언급

### 개선 방향
- 회원가입 시 또는 분석 시 **플랫폼 선택 UI** 제공
- 플랜에 따라 선택 가능 플랫폼 수 제한
  - Free: 1개
  - Plus: 2개
  - Pro: 전체

### UI 흐름
```
분석 페이지 진입
  → 플랫폼 선택 (YouTube / Blog / Instagram)
  → 선택한 플랫폼에 맞는 URL 입력 안내
  → 분석 시작
```

### 플랫폼별 URL 예시
| 플랫폼 | 지원 URL 형식 |
|--------|--------------|
| YouTube | youtube.com/@채널, youtube.com/channel/UC... |
| Blog | blog.naver.com/아이디, 아이디.tistory.com, medium.com/@아이디 |
| Instagram | instagram.com/아이디 |

## 2. 벤치마킹 채널 추천

### 개념
- 분석 결과에 **실제 잘 되고 있는 채널**을 추천
- 피드백 내용이 잘 반영된 실존 채널
- 팩트 기반 (실제 존재하는 채널, 실제 데이터)

### 구현 방식
- YouTube: 같은 카테고리/키워드로 YouTube Data API 검색 → 조회수/구독자 상위 채널 추출
- Blog: 네이버 검색 API로 인기 블로그 추출
- Instagram: 해시태그 기반 인기 계정 (Phase 3)

### 리포트에 표시할 내용
```
📌 참고하면 좋은 채널
┌─────────────────────────────────────┐
│ 채널명: OOO                          │
│ 구독자: 12.3만명                      │
│ 특징: 질문형 제목 + 얼굴 썸네일 활용   │
│ 링크: youtube.com/@...               │
│ 이 채널이 잘하는 점:                   │
│ - 주 3회 꾸준한 업로드               │
│ - 영상 초반 5초 훅이 강력            │
└─────────────────────────────────────┘
```

### 플랜별 제공
| 플랜 | 벤치마킹 |
|------|---------|
| Free | ❌ (모자이크 처리, "Pro에서 확인 가능") |
| Plus | 1개 채널 |
| Pro | 3개 채널 + 상세 비교 |

## 3. 플랜별 데이터 분기 (모자이크)

### 개념
- 무료 사용자에게는 **일부 데이터를 모자이크(잠금)** 처리
- 개발자 도구로도 볼 수 없게 **백엔드에서 아예 데이터를 안 보냄**
- "이런 정보가 있다"는 것만 보여주고 → 결제 유도

### 백엔드 응답 분기
```typescript
// Free 유저 응답
{
  "score": 70,
  "summary": "하담 채널은 음악 콘텐츠를 중심으로...",
  "strengths": ["최근 영상들이..."],  // 1개만
  "weaknesses": ["구독자가 1명..."],  // 1개만
  "actions": ["LOCKED"],              // 잠금
  "contentIdeas": ["LOCKED"],         // 잠금
  "benchmarks": ["LOCKED"],           // 잠금
}

// Plus 유저 응답
{
  "score": 70,
  "summary": "...",
  "strengths": [...],     // 전체
  "weaknesses": [...],    // 전체
  "actions": [...],       // 전체
  "contentIdeas": [...],  // 전체
  "benchmarks": [...],    // 1개
}

// Pro 유저 응답
{
  // 모든 데이터 전체 제공
}
```

### 프론트엔드 표시
- 잠금된 항목: 블러(모자이크) + 자물쇠 아이콘 + "Plus/Pro에서 확인" 버튼
- 클릭 시 → 업그레이드 페이지로 이동

## 4. 마이페이지 / 계정 관리

### 포함 기능
| 기능 | 설명 |
|------|------|
| 프로필 | 이름, 이메일, 가입일, 로그인 방식 |
| 플랜 현황 | 현재 플랜, 남은 분석 횟수, 업그레이드 버튼 |
| 분석 히스토리 | 과거 분석 결과 목록 (날짜, 채널명, 점수) |
| 리포트 열람 | 히스토리에서 클릭 → 리포트 재조회 |
| 라이선스 업그레이드 | 플랜 비교 + 결제 (Phase 2 후반) |
| 문의하기 | 문의 폼 (제목, 내용, 카테고리) |
| 로그아웃 | 세션 종료 |

### 분석 히스토리 열람 정책
| 플랜 | 열람 |
|------|------|
| Free | 생성 후 7일간 |
| Plus/Pro | 구독 중 무제한 |
| 해지 후 | 열람 불가 (재구독 시 복원) |

## 5. 문의하기

### 구현
- 프론트: 문의 폼 (제목, 카테고리, 내용)
- 백엔드: DB에 저장 + (향후) 관리자 이메일 알림
- 관리자 페이지에서 문의 목록 확인 + 답변

### 카테고리
- 결제/환불
- 분석 오류
- 기능 제안
- 기타
