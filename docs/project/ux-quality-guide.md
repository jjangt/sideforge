# UX 개선 & 분석 품질 가이드

## 1. 플랫폼 탭 변경 시 입력 초기화

- 분석 페이지에서 YouTube → Blog 등 탭 전환 시 URL 입력값 초기화
- 사용자가 이전 플랫폼 URL을 실수로 그대로 제출하는 걸 방지

## 2. 새로고침 시 페이지 유지

- 모든 페이지에서 F5 또는 브라우저 새로고침 시 현재 페이지 유지
- Expo Router는 URL 기반이므로 기본적으로 유지됨
- 상태(로그인 등)는 AsyncStorage에 persist 되어 있어 새로고침 후에도 복원
- 리포트 페이지는 URL에 reportId가 있으므로 새로고침 시 API 재호출

## 3. 함수화 원칙

### 규칙
- 모든 UI 요소는 재사용 가능한 컴포넌트로 분리
- 비즈니스 로직은 별도 함수/훅으로 분리
- 특정 페이지에서 변형 필요 시 → 기존 함수를 래핑한 커스텀 함수 생성
- 직접 수정하지 않고 확장하는 방식 (Open/Closed 원칙)

### 예시
```typescript
// 기본 함수
export function formatScore(score: number): string { ... }

// 특정 페이지에서 커스텀 필요 시
export function formatScoreWithEmoji(score: number): string {
  return `${getScoreEmoji(score)} ${formatScore(score)}`;
}
```

- 복잡한 로직에는 인라인 주석
- 파일 상단에 해당 파일의 역할 설명

## 5. 벤치마킹 추천 근거

### 현재 문제
- "크러쉬는 다양한 음악을 합니다" → 왜 추천하는지 근거 없음

### 개선 방향
- 사용자의 약점과 연결하여 추천 이유 명시
- "당신의 채널은 [약점]이 있는데, 이 채널은 [그 약점을 극복한 방법]을 잘 하고 있어서 추천합니다"

### AI 프롬프트에 반영
```
벤치마크 채널 추천 시, 반드시 사용자 채널의 약점과 연결하여 추천 이유를 작성하세요.
형식: "당신의 채널은 [약점]이 있습니다. [추천 채널]은 [약점 극복 방법]을 통해 성공했으므로 참고하세요."
```

## 6. 리포트 시각화

### 현재 문제
- 텍스트만 나열 → 눈에 안 들어옴

### 개선 방향
| 항목 | 시각화 방식 |
|------|-----------|
| 종합 점수 | 원형 게이지 (이미 있음) |
| 영상별 조회수 | 막대 그래프 |
| 조회수 추이 | 라인 차트 |
| 강점/약점 | 레이더 차트 (5각형) |
| 좋아요 비율 | 퍼센트 바 |
| 업로드 빈도 | 캘린더 히트맵 |

### 구현
- react-native에서 사용 가능한 차트: 직접 View로 구현 (외부 라이브러리 최소화)
- 막대 그래프, 프로그레스 바 등은 커스텀 컴포넌트로 제작

## 7. 점수 기준 개선

### 현재 문제
- 구독자 1명, 영상 3개, 평균 조회수 300에 60점 → 기준이 너무 관대

### 개선된 점수 기준

| 항목 | 가중치 | 기준 |
|------|--------|------|
| 구독자 수 | 25% | 0~100: 1000명 미만=낮음, 1만+=높음 |
| 평균 조회수 | 25% | 영상당 평균, 1000회 미만=낮음 |
| 좋아요 비율 | 15% | 조회수 대비 좋아요 % (3% 이상=우수) |
| 업로드 빈도 | 15% | 월 4회 이상=우수, 1회 미만=낮음 |
| 영상 수 | 10% | 10개 미만=초기, 50개+=안정 |
| 성장 추세 | 10% | 최근 영상 조회수가 이전보다 증가? |

### 점수 등급

| 점수 | 등급 | 의미 |
|------|------|------|
| 90~100 | 우수 | 성장세가 뚜렷, 수익화 가능 |
| 70~89 | 양호 | 기본기 갖춤, 개선하면 성장 가능 |
| 50~69 | 보통 | 잠재력 있으나 개선 필요 |
| 30~49 | 미흡 | 전략 수정 필요 |
| 0~29 | 초기 | 시작 단계, 기초부터 |

### 예시 (하담 채널)
- 구독자 1명 → 구독자 점수: 5/25
- 평균 조회수 ~300회 → 조회수 점수: 8/25
- 좋아요 비율 ~1% → 좋아요 점수: 5/15
- 업로드 빈도: 3개/전체 → 빈도 점수: 3/15
- 영상 수 3개 → 영상 점수: 2/10
- 성장 추세 불명 → 성장 점수: 3/10
- **합계: 약 26/100 → "초기" 등급**

이게 훨씬 현실적인 점수.

## 8. AI 프롬프트 점수 기준 반영

AI에게 점수 계산 기준을 명확히 지시:
```
점수 기준:
- 구독자: 1000명 미만=0~20점, 1만명=50점, 10만명=80점, 100만명=100점
- 평균 조회수: 구독자 대비 비율로 판단 (10% 이상이면 양호)
- 업로드 빈도: 주 1회 이상=높음, 월 1회=보통, 그 이하=낮음
- 영상 3개, 구독자 1명, 평균 조회수 300이면 20~30점이 적절합니다.
```
